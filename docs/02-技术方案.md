## 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 框架 | 原生 HTML + CSS + JavaScript（Vanilla JS） | 零依赖、轻量、小白友好、无需构建工具 |
| 存储 | LocalStorage + localStorage.getItem/setItem | 简单够用，10 个场景 50 个物品数据量很小 |
| 图片存储 | Base64 DataURL 存入 LocalStorage | 无需服务器，图片存在本地 |
| 路由 | Hash-based SPA | 无需服务器配置，直接 HTML 打开即可 |
| PWA | manifest.json + Service Worker | iOS 和 Mac 均可添加到桌面，离线可用 |
| 图标 | Emoji + 用户上传图片 | Emoji 零成本，上传图片做补充 |

### 为什么不选框架

- 项目规模小（10 场景 × 5 物品 ≈ 50 条数据）
- 原生 JS 可读性最高，方便以后你自己修改
- 零构建步骤，改完 HTML 刷新就能看到效果
- 没有任何依赖，永不担心版本兼容问题

## 数据模型

### Scene（场景）

```json
{
  "id": "scene_1712345678901",
  "name": "撸铁",
  "icon": "💪",
  "isPinned": false,
  "createdAt": "2026-07-07T10:00:00.000Z",
  "lastUsedAt": null,
  "useCount": 0
}
```

```json
{
  "id": "item_1712345678902",
  "sceneId": "scene_1712345678901",
  "name": "健身手套",
  "icon": "🧤",

  "storageLocation": "玄关鞋柜第三层",
```

## 数据流

```
用户操作 → JS 更新内存数据 → 同步写入 LocalStorage → 重新渲染 UI
                     ↑
          页面刷新后从 LocalStorage 读取恢复
```

## 应用结构（SPA）

```
index.html
├── 首页（场景列表）
│   ├── 搜索栏
│   ├── 预设场景 + 自定义场景
│   └── 新建场景按钮
├── 场景详情页
│   ├── 场景名称 + 频次信息
│   ├── 物品清单（复选框）
│   └── 添加物品按钮
├── 物品详情页
│   ├── 物品名称
│   ├── 照片展示 / 上传
│   ├── 图标选择
│   └── 存放位置编辑
└── 成功出门弹窗
```

## PWA 策略

- manifest.json：配置名称、图标、主题色、display: standalone
- Service Worker：缓存所有静态资源，实现离线可用
- iOS 上通过"添加到主屏幕"获得全屏原生体验

---

## 安全性

- 所有数据存储在用户本地浏览器
- 不上传任何数据到服务器
- 用户可自行通过浏览器开发者工具导出数据
