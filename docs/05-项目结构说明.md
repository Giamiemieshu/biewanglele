## 文件职责说明

### HTML / CSS

| 文件 | 职责 |
|------|------|
| `index.html` | 单页应用入口，包含所有页面的 DOM 结构（通过 JS 控制显示隐藏） |
| `css/style.css` | 全部样式，遵循设计规范 `docs/03-设计规范.md` |

### JavaScript 模块

| 文件 | 职责 |
|------|------|
| `js/store.js` | `DataStore` 类：封装 LocalStorage 读写，提供 get/set/update/delete |
| `js/app.js` | 路由控制、页面切换、初始化、挂载事件监听 |
| `js/scenes.js` | 场景列表渲染、新建/删除/置顶、频次统计 |
| `js/items.js` | 物品清单渲染、添加/删除、复选框逻辑 |
| `js/item-detail.js` | 物品详情编辑（照片上传、图标选择、存放位置） |
| `js/checkout.js` | 检查全部勾选、成功出门弹窗、重置状态 |
| `js/search.js` | 搜索输入监听、实时过滤 |

## AI / 开发者工作指引

如果你是 AI 助手首次接手此项目，请按以下顺序了解项目：

1. 阅读 `docs/01-需求文档.md` — 了解产品要做什么
2. 阅读 `docs/02-技术方案.md` — 了解技术选型和数据模型
3. 阅读 `docs/03-设计规范.md` — 了解 UI 设计标准
4. 阅读 `docs/04-开发步骤.md` — 了解当前开发进度
5. 阅读 `devlog/DEVLOG.md` — 了解已完成的工作
6. 阅读最新的 `devlog/YYYY-MM-DD.md` — 了解最近的工作和待办

### 开发规范

- 每个 Step 完成后更新 `devlog/` 记录
- CSS 变量统一在 `:root` 中定义，参考设计规范
- JS 使用 ES6 语法，保持函数简洁
- 所有文本使用中文
- 代码注释遵循"为什么这么做"而非"做了什么"

### 上下文管理

如果 AI 上下文过长：
- 优先关注当前 Step 的任务
- 通过 devlog 和 docs 快速恢复记忆
- 不要一次性改动过多文件
