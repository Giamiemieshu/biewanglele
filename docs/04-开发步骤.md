# 04-开发步骤

> 文件名：`docs/04-开发步骤.md`
> 最后更新：2026-07-07
> 
> 本文档规划了"别忘带了"项目的所有开发阶段。
> 每个 Step 完成后，在 devlog 中记录。
> 避免一次性改动过多，每一步完成后再开始下一步。

---

## 总览

| 步骤 | 内容 | 预计复杂度 |
|------|------|-----------|
| Step 1 | 数据模型 + LocalStorage 持久层 | ⭐ |
| Step 2 | 场景管理（CRUD + 置顶 + 频次） | ⭐⭐ |
| Step 3 | 物品清单（CRUD + 复选框） | ⭐⭐ |
| Step 4 | 物品详情页（照片、图标、存放位置） | ⭐⭐ |
| Step 5 | 成功出门流程 + 弹窗 + 状态重置 | ⭐⭐ |
| Step 6 | 搜索功能 | ⭐ |
| Step 7 | PWA 配置（manifest + Service Worker） | ⭐ |
| Step 8 | UI 打磨 + 空状态 + 动画 + 测试 | ⭐⭐ |

---

## Step 1：数据模型 + LocalStorage 层

**目标：** 定义数据结构、实现数据读写工具函数。

**文件：** `js/store.js`

**交付物：**
- [ ] 定义 Scene / Item / CheckoutRecord 数据结构
- [ ] 实现 DataStore 类（get、set、update、delete）
- [ ] 预设场景种子数据
- [ ] 单元验证：在浏览器控制台可调用数据 API

---

## Step 2：场景管理

**目标：** 展示场景列表、新建/删除/置顶场景、记录频次。

**文件：** `js/scenes.js`, `index.html`（首页）

**交付物：**
- [ ] 首页渲染场景卡片列表
- [ ] 新建场景（输入名称 + 选择图标）
- [ ] 删除场景（确认弹窗）
- [ ] 置顶/取消置顶
- [ ] 显示使用频次和最近使用日期

---

## Step 3：物品清单

**目标：** 在场景详情页中展示物品清单、添加/删除物品、复选框勾选。

**文件：** `js/items.js`, `index.html`（场景详情页）

**交付物：**
- [ ] 场景详情页展示物品列表
- [ ] 添加物品（名称 + 图标）
- [ ] 删除物品
- [ ] 复选框勾选/取消
- [ ] 物品排序

---

## Step 4：物品详情页

**目标：** 点击物品进入详情，编辑照片、图标、存放位置。

**文件：** `js/item-detail.js`, `index.html`（物品详情页）

**交付物：**
- [ ] 物品详情页展示
- [ ] 上传照片（FileReader → DataURL）
- [ ] 选择图标（Emoji 选择器）
- [ ] 编辑存放位置
- [ ] 保存修改

---

## Step 5：成功出门流程

**目标：** 全部勾选 → 成功出门弹窗 → 重置勾选状态。

**文件：** `js/checkout.js`

**交付物：**
- [ ] 检测所有物品是否已勾选
- [ ] 成功出门弹窗（鼓励文案 + 动画）
- [ ] 记录出门时间到 CheckoutRecord
- [ ] 更新场景使用频次
- [ ] 重置所有复选框（下次进入时）

---

## Step 6：搜索功能

**目标：** 全局搜索场景和物品名称。

**文件：** `js/search.js`

**交付物：**
- [ ] 首页搜索框
- [ ] 输入时实时过滤场景
- [ ] 搜索结果可点击进入

---

## Step 7：PWA 配置

**目标：** 配置 manifest.json + Service Worker，实现桌面端安装和离线缓存。

**文件：** `manifest.json`, `sw.js`

**交付物：**
- [ ] manifest.json（应用名、图标、主题色）
- [ ] Service Worker（缓存策略）
- [ ] iOS 添加到主屏幕测试

---

## Step 8：UI 打磨

**目标：** 完善交互细节、动画、空状态、边界情况。

**交付物：**
- [ ] 过渡动画（页面切换、添加/删除）
- [ ] 空状态提示
- [ ] 错误处理
- [ ] 数据导出/备份
- [ ] 最终测试

---

## 原则

1. **每次只做一步**，完成后再开始下一步
2. **每一步完成时**，更新 `devlog/` 中的记录
3. **每一步完成后**，用户确认后再继续
4. **避免上下文过长**，如果某一步需要大量代码，考虑拆分子步骤
